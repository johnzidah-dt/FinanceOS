# Parcours utilisateurs

## 1. Creation d'une facture

1. L'agent choisit ou cree un client.
2. Il renseigne l'objet, la reference, la date et les lignes.
3. Il choisit un projet existant ou laisse le systeme creer un nouveau projet.
4. Le systeme calcule les totaux.
5. La facture est generee selon le modele fourni.
6. La facture passe en statut emise.

## 2. Paiement d'une facture

1. L'utilisateur ouvre la facture.
2. Il saisit le montant paye.
3. Il choisit le moyen de paiement.
4. Il choisit la destination : caisse ou banque.
5. Le systeme met a jour le solde de la destination.
6. Le statut facture devient payee, partiellement payee ou en attente.

## 3. Suivi quotidien de caisse

1. Le caissier consulte le solde initial.
2. Il voit les encaissements du jour.
3. Il enregistre ou valide les sorties.
4. Le systeme affiche le solde temps reel.
5. Le caissier exporte le journal de caisse.

## 4. Decaissement

1. Un utilisateur cree une demande : beneficiaire, motif, projet, montant.
2. Un responsable valide la demande.
3. Le systeme verifie la liquidite disponible.
4. Le decaissement est marque comme decaisse.
5. Une fiche signable est generee.
6. Le mouvement de tresorerie est enregistre.

## 5. Reporting

1. Le manager choisit une periode : jour, mois ou temps reel.
2. Il filtre par client, projet, caisse ou banque.
3. Le systeme affiche les indicateurs financiers.
4. Il exporte le rapport en Excel ou Word.

## 6. Pilotage par projet

1. Le manager ouvre un projet ou evenement.
2. Il consulte les factures rattachees.
3. Il consulte les paiements recus.
4. Il consulte les depenses et decaissements.
5. Le systeme affiche la marge estimee.
